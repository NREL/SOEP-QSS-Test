ZCBoolean2 Model QSS Notes

The QSS runs used the Dymola FMU but the JModelica FMU gives the same results.
(PyFMI is not simulating these FMUs correctly.)

The QSS runs use dtMax to avoid deactivation:
QSS ZCBoolean2.fmu --dtMax=0.001 --qss=<QSS_method>

Notes
 x is linear between ZC events so it deactivated and exact in QSS2+ and doesn't need dtInf
 u doesn't have causality="input" in xml: It is an "internal" variable
  Ignore u since it isn't discontinuous so z1/2 integration should be OK without observer updates
  Internal variables with discontinuous functions or discrete events need to be input functions for QSS to operate correctly
 z1/2 have zero 2nd derivatives at t=0 giving large (14s) requantization time (infinity with exact derivatives)
  This is an example of what "near-deactivation": dtInf won't kick in: Need dtMax with current QSS
